# Contributing to QualityTrack WP

Thank you for your interest in this project. QualityTrack WP is currently a proprietary product demo maintained by a single developer. Contributions are welcome in specific forms as described below.

---

## What You Can Contribute

### Bug Reports
If you find something broken — a button that doesn't respond, a layout issue on a specific device, or a logic error in the workflow — please open a GitHub Issue with:

- A clear description of the problem
- The browser and device you are using
- Steps to reproduce the issue
- A screenshot if relevant

### Suggestions & Feature Requests
If you have ideas for improving the demo experience or the planned production application, open a GitHub Issue with the label `enhancement`. Please describe:

- The problem you're trying to solve
- Your proposed solution
- Any relevant context from the construction / real estate domain

### Domain Expertise
This system is built for the Indian real estate and construction industry. If you are a site engineer, QC inspector, project manager, or developer with domain knowledge and you notice that any checklist item, workflow step, or terminology is technically incorrect — please raise an Issue. Domain accuracy is critical for this product.

---

## What This Project Does Not Accept

- Pull requests that change the core workflow logic without prior discussion
- Contributions that alter the single-file architecture (`index.html`)
- Design changes to colours, typography, or branding
- Any contribution that introduces external JavaScript dependencies

---

## How to Open an Issue

1. Go to the **Issues** tab in this repository
2. Click **New Issue**
3. Choose the appropriate template (Bug Report / Feature Request)
4. Fill in the details clearly and submit

---

## Development Setup

No build tools or dependencies are needed.

```bash
git clone https://github.com/iammohith/Waterproofing-Activity-Management-System-Demo.git
cd Waterproofing-Activity-Management-System-Demo
open index.html   # macOS
# or
start index.html  # Windows
# or
xdg-open index.html  # Linux
```

The entire application is a single self-contained HTML file. All changes should be made to `index.html` only.
